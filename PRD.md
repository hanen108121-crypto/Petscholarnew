# PRD (Product Requirements Document) - 北科遊戲化學業交流區

## 1. 專案背景與定位
在大學校園中，跨學科的學術討論與同儕解答是學習的重要一環。然而，傳統討論區（如 Dcard、校內論壇）往往缺乏足夠的誘因鼓勵學生積極回答他人的課業難題。
**北科遊戲化學業交流區** 是一個專為課堂展示與期末報告設計的前端展示 Demo。本系統結合了**論壇討論**與**遊戲化虛擬寵物養成**機制：學生透過在論壇解答課業問題獲得「金幣」，金幣可以用於商店購買食物，餵食虛擬寵物「北科科」以恢復其「生命值（HP）」，藉此建立一個「同儕互助、答題養成」的社群氛圍。
本專案為**純靜態網頁展示**，不需要安裝任何套件，也無需連接後端或資料庫。

## 2. 目標使用者與場景
- **求助學生**：在學習（如微積分、演算法）遇到難題時，於對應的科系看板發文，並提供金幣懸賞。
- **解答學霸**：利用自身專業解答問題，當答案被採納時獲得金幣，用來照顧自己瀕臨飢餓的虛擬寵物。
- **簡報展示者**：能在 3 分鐘內透過預設情境，流暢地向台下師生展示「進入看板 -> 解答問題 -> 獲得金幣 -> 商店購物 -> 寵物恢復狀態」的完整流程，並清晰說明 Object、Array 與 Tree 的資料結構使用。

## 3. 技術規格與限制
- **前端核心**：純 HTML5, CSS3, Vanilla JavaScript (ES6)。
- **無後端/無資料庫**：所有資料儲存於靜態 JavaScript 檔案中（`data/boardsData.js` 與 `data/userData.js`）。
- **零依賴安裝**：使用者只需在瀏覽器中開啟 `index.html` 即可完整執行，不得使用 Node.js, npm, React, Vue 等框架。
- **資料結構聚焦**：以網頁動畫與 SVG 視覺化圖表展示資料結構運作（特別是留言樹狀 Tree）。

## 4. 功能需求 (Functional Requirements)

### F1. 跨科系看板與文章總覽 (Department Boards & Threads)
- 提供四大科系看板切換：電機系（Blue）、資財系（Purple）、資工系（Green）、機械系（Orange）。
- 各看板首頁顯示文章列表，包括：文章標題、發文者、懸賞金幣、發問時間、狀態（已解決/未解決）、標籤。
- 提供「今日熱門發問」與「學術英雄榜」等簡報展示用的靜態看板小工具。

### F2. 互動式 Q&A 與解答採納 (Thread View & Adoption)
- 點擊文章進入詳細閱讀頁面，呈現完整的提問內容（如微積分公式或程式碼）。
- 留言區使用**樹狀結構 (Tree)** 渲染，支援多層級回覆（即對某個回覆再進行回覆）。
- 模擬發言：使用者可點擊「我來解答」按鈕，輸入名字與回答，送出後即時插入到留言樹中，且畫面即時重新渲染。
- 模擬採納：在未解決的文章中，針對特定解答點擊「採納解答」，觸發金幣飛入動畫，金幣數增加（+50 點），寵物冒出開心的對話氣泡，文章狀態更新為「已解決」。

### F3. 虛擬寵物狀態區與養成商店 (Virtual Pet & Item Shop)
- 寵物「北科科」位於側邊欄（或左側固定區），具備呼吸與隨機眨眼的 SVG 動畫。
- 顯示狀態條：
  - **生命值 (HP)**：滿分 100，預設為 60（疲憊狀態）。低於 40 時寵物顯示無精打采的表情。
  - **金幣 (Coins)**：預設 30。可用於商店消費。
  - **經驗值 (EXP) 與等級**：答題或餵食可以獲得經驗值，滿 100 經驗值升級。
- 商城面板（或商店分頁）提供商品購買：
  - **學霸雞排**（15 金幣，恢復 20 HP）
  - **爆肝咖啡**（20 金幣，恢復 30 HP）
  - **期末歐趴糖**（50 金幣，恢復 100 HP）
- 點擊購買時，扣除金幣，播放寵物「吃東西」動畫與愛心上升特效，生命值（HP）對應增加。若金幣不足，畫面給出紅字警告，不扣款也不恢復。

### F4. 資料結構解析與樹狀圖視覺化 (Data Structure Visualizer)
- 專門的說明區，解釋三大資料結構的運用：
  - **Object**：以 key-value 存儲各科系看板與寵物資訊。
  - **Array**：儲存文章列表與商店商品清單，展示資料遍歷。
  - **Tree**：儲存嵌套留言，展示對話分支。
- **動態留言 SVG 樹狀圖**：將當前文章的留言資料動態繪製成 Node-Link 樹狀圖。點擊網頁上的某條留言，SVG 樹中的對應節點會同步高亮，直觀展示何謂樹的「節點」、「子節點」與「根節點」。

## 5. 非功能需求 (Non-Functional Requirements)
- **極致視覺（Premium Aesthetics）**：採用現代深色玻璃擬物化（Glassmorphism）風格，提供高對比霓虹色系，符合簡報高吸睛度。
- **穩定性與防呆**：當未輸入留言內容就點擊發言時，給予輸入框震動或提示，而非報錯；金幣不足時購買商店道具應有提示音或文字提示。
- **簡報輔助模式**：首頁提供「3 分鐘簡報導覽」按鈕，點擊後會顯示半透明指示圈，引導講者一步步點擊，確保現場 Demo 100% 成功。
