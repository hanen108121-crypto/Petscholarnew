# 北科遊戲化學業交流區 (Gamified Academic Exchange Zone - Static Demo)

本專案是一個專為課堂展示與期末報告設計的「遊戲化校園學業 Q&A 交流平台」前端 Demo。
它結合了**論壇討論**與**虛擬寵物養成**：使用者透過解答其他同學的課業問題賺取金幣，並可用金幣在商店購買食物餵食虛擬寵物「北科科」，恢復其生命值。本系統以純前端靜態頁面實作，無需後端與資料庫，並提供豐富的資料結構互動解析（特別是留言樹狀圖的 SVG 視覺化）。

---

## 🌟 核心特色
1.  **現代深色霓虹視覺**：採用現代深色玻璃擬物化 (Dark Glassmorphism) 設計，帶來吸睛的前端介面。
2.  **虛擬寵物互動中心**：左側整合「北科科」SVG mascot，具備動態表情變化（疲倦/開心/吃東西/正常）與即時狀態條。
3.  **多層級留言樹狀結構**：留言板支援多層級巢狀回覆（對回覆進行回覆），完美重現樹狀資料關係。
4.  **互動式金幣與商店系統**：模擬採納解答獲得金幣，並能在商店一鍵購買道具餵食寵物，數值即時變更並伴隨 UI 特效。
5.  **留言樹 SVG 視覺化解析**：實時將留言板渲染為 Node-Link 樹狀圖，點擊留言高亮對應樹節點，生動演繹 Tree 資料結構。
6.  **3 分鐘簡報導覽模式**：專為簡報設計的引導系統，一步步提示點擊，確保 Demo 順暢不卡頓。

## 📊 資料結構答詢指引

在進行期末報告或面對教授提問時，可以參考以下回答：

### 1. 使用了哪幾種資料結構？
- **Object (物件)**：
  - 看板資料庫：使用以看板 ID 為 Key 的 Object 儲存，例如 `{ ee: { name: '電機系', posts: [...] } }`，方便 $O(1)$ 快速查找看板。
  - 寵物與使用者狀態：用一個 Object 記錄寵物的當前生命值、金幣與等級。
- **Array (陣列)**：
  - 看板內的文章清單、商店中的商品列表、以及樹狀節點中的 `replies` 子節點陣列，都是使用 Array 儲存，方便進行遍歷與增刪。
- **Tree (樹)**：
  - 用於儲存留言回覆。每個留言節點都是一個樹的節點，包含：`id`, `author`, `content`, `timestamp`, 以及子節點陣列 `replies`。

### 2. 為什麼這裡適合用這些資料結構？
- **討論區的留言**本質上就是一種樹狀結構：一篇文章有數個主留言，每個主留言又可以有子回覆，子回覆下還可以有子子回覆。使用 **Tree** 結構能夠精確表達這種階層與脈絡關係，避免使用扁平 Array 導致回覆關係錯亂。
- **看板列表與商品列表**的長度在展示中是固定的或線性增減的，使用 **Array** 最易於進行渲染與依序顯示。
- **看板查找**使用 **Object** 對照表（以 ID 為 key），能以 $O(1)$ 的效率直接加載特定科系的看板內容，不需要掃描整個看板陣列。

### 3. 這個資料結構如何影響功能運作？
- **樹狀留言渲染**：系統透過遞迴深度優先遍歷（DFS）對留言樹進行遍歷，將其渲染成嵌套的 HTML 結構（`<ul>` 與 `<li>` 的嵌套）。
- **SVG 樹狀圖繪製**：系統在「資料結構解析」分頁中，讀取同一份留言 Tree 資料，動態計算每個節點的 (X, Y) 軸座標，繪製出帶有霓虹連線的 SVG 樹狀節點圖。當使用者點擊 HTML 的某一則留言，系統利用 ID 尋找 Tree 中的節點，並高亮 SVG 中對應的 SVG 圓圈，達到了資料結構與 UI 的雙向綁定。
